.Net 平台的数据分析解决方案

发表于2023-03-01|更新于2023-03-12|C#数据分析
|字数总计:2.3k|阅读时长:10分钟|阅读量:

背景

因个人在工作中有数据分析相关的工作, 也因此学习了 Pandas 库进行数据分析. 目前工程已经较大, 且具有将程序打包给别人使用的需求. 但在 Python 环境下对体量稍大的项目的维护成本很高, 打包经常出问题且包体很大. 于是我转而学习 C#.

在 C# 里也发现了一个神级表格库 MiniExcel. 这是一个访问效率极快, 内存占用极低, 专注与数据而不在乎样式的表格库, 也解决了 Openpyxl 对于大量空行空列 OOM 的问题. 但没有 Pandas 一直是我的心病, 在了解到 Microsoft.Data.Analysis.DataFrame 和 SciSharp 下的项目都无法满足需求后我转而研究 C# 调用 Python 库.

我找到了 IronPython 和 Pythonnet . IronPython 无法调用 Pandas 被我直接排除. Pythonnet 虽然可以, dynamic 的数据类型, 数据交互的不支持类型, 还有导入 Pandas 库就上百兆的内存占用依然让我是苦不堪言. 而这次要求制作百页的数据 PPT, 让我知道我必须要完成 PPT 的自动化生成了. 于是我完全放弃 Python, 回到了当初早就相遇但未曾相识的 Deedle 的怀抱.

在此, 我宣布 Deedle 的 .Net 平台里唯一一个可用的数据分析库.

一些坑

  • 填充空缺值需要填充的值与数据帧里的值类型一致, 如果是 decimal 就只能写 decimal , Deedle 不会帮忙做任何的类型转换.
  • 在 decimal 除以 0 的时候会显示为 ∞, 但在 decimal 中没有无穷大的定义, float double 却都有. 查询后发现 Deedle 已经将 decimal 类型转变为 double 类型了.
